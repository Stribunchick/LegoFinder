# Установка и запуск рабочей версии

Этот файл описывает установку и запуск версии проекта, которая запускается через `app.py` и использует `robust_pipeline`.

## Что нужно для запуска

- Windows 10 или Windows 11
- Python 3.12 или Python 3.13
- веб-камера, если нужен режим работы с камерой

Важно:

- для текущей рабочей версии достаточно пакетов из `requirements.txt`;
- экспериментальные части проекта в эту инструкцию не входят;
- приложение проверялось на Python `3.13`, `opencv-python 4.12.0.88`, `numpy 2.2.6`, `PySide6 6.9.2`.

## 1. Установить Python

1. Установите Python 3.12 или 3.13 с официального установщика.
2. Во время установки включите опцию `Add Python to PATH`.
3. После установки откройте PowerShell и проверьте:

```powershell
python --version
```

Ожидается версия не ниже `3.12`.

## 2. Перейти в папку проекта

Откройте PowerShell в корне проекта:

```powershell
cd LegoFinder
```

## 3. Создать виртуальное окружение

Рекомендуется ставить зависимости в отдельное виртуальное окружение:

```powershell
python -m venv .venv
```

## 4. Активировать виртуальное окружение

Для PowerShell:

```powershell
.venv\Scripts\Activate.ps1
```

Если PowerShell запрещает запуск скриптов, выполните:

```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
```

После этого снова выполните:

```powershell
.venv\Scripts\Activate.ps1
```

После активации слева в строке терминала обычно появляется префикс `(.venv)`.

## 5. Обновить pip

```powershell
python -m pip install --upgrade pip
```

## 6. Установить зависимости

Установите только зависимости рабочей версии:

```powershell
pip install -r requirements.txt
```

Файл `requirements.txt` для рабочей версии содержит:

- `numpy==2.2.6`
- `opencv-python==4.12.0.88`
- `PySide6==6.9.2`

## 7. Запустить приложение

Из корня проекта выполните:

```powershell
python app.py
```

После запуска откроется главное окно приложения.

## 8. Минимальный сценарий первого запуска

1. Нажмите `Добавить деталь`.
2. Сделайте снимок детали или загрузите изображение детали.
3. Эталон лучше подготавливать на однотонном фоне.
4. Нажмите `Обработать изображение`.
5. Вернитесь в главное окно и выберите сохранённую деталь из списка.
6. Нажмите `Старт/Стоп` для запуска камеры или загрузите изображение/видео.

## Где хранятся эталоны

Эталоны рабочей версии сохраняются автоматически в папку:

```text
data/robust_templates
```

Если папки ещё нет, приложение создаст её при работе.

## Что делать, если приложение не запускается

### Ошибка `ModuleNotFoundError`

Скорее всего, не активировано виртуальное окружение или зависимости не установлены. Повторите:

```powershell
.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

### Ошибка из-за версии Python

Проект использует `from typing import override`, поэтому нужен Python `3.12+`.

Проверьте:

```powershell
python --version
```

### Не открывается камера

- убедитесь, что камера не занята другим приложением;
- проверьте разрешения Windows на доступ к камере;
- в рабочем проекте по умолчанию используется камера с индексом `0`.

### Окно запускается, но детекция не работает

- сначала добавьте хотя бы один эталон;
- убедитесь, что нужная деталь выбрана в списке;
- для эталона используйте изображение на однотонном фоне;
- при плохом освещении или сильном смазе качество детекции падает.

## Быстрая проверка после установки

Если зависимости установились без ошибок и команда ниже открывает GUI, значит рабочая версия готова:

```powershell
python app.py
```

## Коротко

# Описание работы проекта



## 1. Общая схема

Проект состоит из двух частей:

- GUI и инфраструктура приложения;
- алгоритмический модуль `robust_pipeline`.

GUI отвечает за:

- запуск окна;
- выбор источника кадров;
- выбор эталона;
- добавление новых деталей;
- отображение результата.

`robust_pipeline` отвечает за:

- сохранение эталона;
- загрузку эталона;
- обработку кадра;
- поиск детали;
- расчёт `confidence`;
- возврат `bbox`.

## 2. Основные модули

- `app.py`  
  Точка входа. Создаёт `QApplication`, включает оптимизации OpenCV, запускает `MainWindow`.

- `main_window.py`  
  Главное окно приложения.

- `add_part_window.py`  
  Окно добавления нового эталона детали.

- `application/appcontroller.py`  
  Связывает GUI, захват кадров и обработку.

- `application/frame_grabber.py`  
  Периодически читает кадры из текущего источника.

- `application/frame_processor.py`  
  Передаёт кадры в `robust_pipeline`, хранит только последний кадр, рисует результат.

- `robust_pipeline/reference_manager.py`  
  Сохраняет эталон на диск и загружает его обратно.

- `robust_pipeline/detector.py`  
  Основной алгоритм поиска детали на кадре.

- `robust_pipeline/visualization.py`  
  Рисует `bbox` и подпись на изображении.

## 3. Как работает приложение

После запуска:

1. создаётся `MainWindow`;
2. создаётся `AppController`;
3. `AppController` поднимает два рабочих объекта в отдельных потоках:
   - `FrameGrabber`
   - `FrameProcessor`
4. GUI через сигналы управляет источником кадров, порогом уверенности и выбранной деталью.

## 4. Как идёт поток кадров

Источником может быть:

- камера;
- изображение;
- видео.

Путь кадра такой:

`Source -> FrameGrabber -> FrameProcessor -> RobustPartMatchingPipeline -> draw_detection -> MainWindow`

`FrameGrabber` читает кадры по таймеру.  
`FrameProcessor` не накапливает очередь, а держит только последний актуальный кадр. Это уменьшает задержку.

## 5. Как добавляется эталон

В окне `AddPartWindow` пользователь:

1. делает снимок с камеры или загружает изображение;
2. вводит имя детали;
3. запускает сохранение эталона.

Дальше `RobustReferenceManager`:

1. выделяет объект на однотонном фоне;
2. строит маску детали;
3. обрезает изображение по маске;
4. при необходимости уменьшает эталон;
5. считает цветовые, геометрические и локальные признаки;
6. сохраняет всё в папку:

`data/robust_templates/<имя_детали>`

Сохраняются:

- `template.png`
- `mask.png`
- `edges.png`
- `hist.npy`
- `lab_stats.npy`
- `views.npz`
- `meta.json`

## 6. Как устроен эталон

При сохранении эталона строятся:

- маска объекта;
- цветовая гистограмма;
- статистика в LAB;
- карта границ;
- набор synthetic affine-view представлений.

Для каждого affine-view извлекаются локальные признаки:

- `SIFT`, если доступен;
- иначе `AKAZE`.

Для `SIFT` дополнительно применяется `RootSIFT`.

## 7. Как загружается эталон

При выборе детали:

1. `FrameProcessor` вызывает `pipeline.load_reference(...)`;
2. `RobustReferenceManager` загружает данные эталона с диска;
3. детектор сбрасывает tracking;
4. выполняется `warmup`, чтобы уменьшить задержку первого кадра.

После этого эталон готов к поиску на видеопотоке.

## 8. Как работает детекция на кадре

Основная логика находится в `RobustPartDetector.detect(...)`.

### Шаг 1. Подготовка кадра

- кадр уменьшается до рабочего размера;
- строится grayscale-версия с CLAHE;
- при наличии прошлой детекции прогнозируется ROI.

### Шаг 2. Построение маски кандидатов

Детектор строит цветовую маску на основе эталона, используя:

- HSV;
- обратную проекцию гистограммы;
- LAB-хрому.

Это даёт область, где объект вероятнее всего находится.

### Шаг 3. Быстрый поиск

Если есть ROI или хорошая цветовая маска, детектор сначала запускает более дешёвые ветки:

- contour-based candidate;
- fallback `color + shape`.

Если они дают уверенный результат, тяжёлый поиск можно не запускать.

### Шаг 4. Feature matching

Если быстрых веток недостаточно, включается поиск по локальным признакам:

1. извлекаются keypoints и descriptors на кадре;
2. сравниваются с несколькими лучшими `view` эталона;
3. для совпадений считается `homography`;
4. по спроецированной маске получается `bbox`.

### Шаг 5. Оценка качества

Для кандидатов считаются оценки:

- по признакам;
- по покрытию эталона;
- по границам;
- по цвету;
- по внешнему виду;
- по форме маски.

Из них собирается итоговый `confidence`.

### Шаг 6. Возврат результата

Если `confidence` выше порога:

- обновляется tracking;
- возвращается `DetectionResult` с:
  - именем детали;
  - confidence;
  - `bbox`;
  - полигоном;
  - debug-информацией.

Если кандидат слабый, возвращается `None`.

## 9. Как работает tracking

Tracking здесь лёгкий и встроенный в детектор.

После успешной детекции сохраняются:

- имя детали;
- нормализованный `bbox`;
- число пропусков.

На следующих кадрах это позволяет:

- искать сначала рядом с прошлым положением;
- уменьшать область поиска;
- работать на меньшем разрешении;
- реже запускать тяжёлый feature matching.

Если объект несколько кадров не найден, tracking сбрасывается.

## 10. Как рисуется результат

После детекции `FrameProcessor` вызывает `draw_detection(...)`.

Если объект найден:

- рисуется зелёный `bbox`;
- рисуется подпись:

`имя детали | confidence%`

Если объект не найден, кадр показывается без изменений.

## 11. Какие оптимизации уже есть

На уровне приложения:

- отдельные потоки для захвата и обработки;
- обработка только последнего кадра;
- использование таймера с малым интервалом;
- включённые оптимизации OpenCV.

На уровне пайплайна:

- уменьшение рабочего размера кадра;
- отдельный более быстрый режим при tracking;
- ограничение числа `view`;
- ограничение числа признаков на эталоне;
- цветовая локализация до feature matching;
- быстрые contour-ветки до тяжёлого поиска;
- редкий глобальный refresh вместо полного поиска на каждом кадре.

## 12. Короткая схема

### Запуск

`app.py -> MainWindow -> AppController -> FrameGrabber + FrameProcessor`

### Сохранение эталона

`AddPartWindow -> ReferenceManager -> mask -> crop -> features -> save to data/robust_templates`

### Детекция

`frame -> resize -> color mask -> ROI/tracking -> quick candidates -> feature matching if needed -> confidence -> bbox -> draw`

## 13. Итог

Текущая версия проекта работает как realtime-приложение, в котором:

- кадры захватываются отдельно от GUI;
- обработка идёт в отдельном потоке;
- эталон заранее кодируется в компактное представление;
- поиск выполняется через комбинацию:
  - цвета;
  - формы;
  - локальных признаков;
  - гомографии;
  - итогового confidence-скоринга.

Это позволяет одновременно поддерживать:

- интерактивную работу через GUI;
- добавление эталонов из интерфейса;
- поиск детали на изображении, видео или с камеры.
